---
title: "Organize documents into projects in Mike"
description: "Create projects in Mike to group related legal documents, collaborate with teammates, and run AI chats scoped to all project documents."
---

Projects let you group related documents together, share them with colleagues, and run AI chats that can reference every document in the project at once. A typical use case is creating one project per matter or transaction and adding all the relevant agreements, schedules, and ancillary documents to it.

<Tip>
  Mike projects have an optional **CM number** field for matter numbering. If your firm uses a client-matter numbering system, fill in this field when creating or editing a project so it appears alongside the project name across the UI.
</Tip>

## Creating a project

<Steps>
  <Step title="Navigate to Projects">
    Click **Projects** in the sidebar.
  </Step>
  <Step title="Click New project">
    Click the **New project** button in the top-right corner.
  </Step>
  <Step title="Fill in the details">
    Enter a **name** for the project (required). Optionally add a **CM number** for matter tracking. You can also invite collaborators immediately by entering their email addresses in the **Share with** field.
  </Step>
  <Step title="Create">
    Click **Create**. The project opens immediately and is ready to accept documents.
  </Step>
</Steps>

## Adding documents to a project

<Tabs>
  <Tab title="Upload directly">
    Open the project, click **Upload**, and select one or more PDF, DOCX, or DOC files. The documents are processed and attached to the project in the same step.
  </Tab>
  <Tab title="Move a standalone document">
    If you already uploaded a document as a standalone file, you can move it into a project without re-uploading.

    <Steps>
      <Step title="Open the document">
        Navigate to **Documents** and open the document you want to move.
      </Step>
      <Step title="Add to project">
        Click **Add to project** in the document options menu and select the target project.
      </Step>
    </Steps>

    The document's full version history moves with it.
  </Tab>
</Tabs>

## Folder organization

Inside a project you can create folders and subfolders to organize documents into logical groups — for example, by document type or by transaction party.

<Steps>
  <Step title="Create a folder">
    Inside the project, click **New folder** and enter a name.
  </Step>
  <Step title="Move documents into folders">
    Drag a document onto a folder, or open the document options menu and choose **Move to folder**.
  </Step>
  <Step title="Create subfolders">
    Open a folder and click **New folder** again to nest a folder inside it. Mike supports arbitrarily deep folder hierarchies.
  </Step>
  <Step title="Rename or move a folder">
    Click the folder options menu to rename it or move it to a different parent folder.
  </Step>
</Steps>

Deleting a folder moves any documents it contained back to the project root before the folder is removed. No documents are deleted.

## Sharing and collaboration

Each project has a **shared with** list of email addresses. Anyone on that list can view all the project's documents and chats.

<Steps>
  <Step title="Open project settings">
    Click the **People** icon or navigate to the project's **Settings** tab.
  </Step>
  <Step title="Add collaborators">
    Enter one or more email addresses and click **Share**. Collaborators receive access immediately — they do not need to accept an invitation.
  </Step>
  <Step title="Remove collaborators">
    Click the **✕** next to any email address in the people list to revoke their access.
  </Step>
</Steps>

Collaborators can view and chat with all documents in the project. Only the project owner can rename the project, change the CM number, or modify the collaborator list.

## Project chats

Every AI chat created inside a project is scoped to that project. The assistant can read all documents in the project when you ask a question — you do not need to attach files manually.

Project chats are visible to all collaborators, not just the person who started the chat. This means teammates can pick up a conversation and continue it, or review the AI's reasoning on a previous question.

To start a project chat, open the project and click **New chat** in the chats tab.

## Project listing

The main **Projects** page shows every project you own plus any projects shared with you. For each project you can see:

| Metric | Description |
|--------|-------------|
| **Documents** | Number of documents currently in the project |
| **Chats** | Number of AI chat sessions created in the project |
| **Reviews** | Number of tabular reviews linked to the project |

Use these counts to quickly gauge activity across matters without opening each project individually.
