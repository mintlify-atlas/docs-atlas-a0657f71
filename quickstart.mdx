This quickstart walks you through cloning the Mike repository, configuring your environment variables, running the database migration, and launching both services locally so you can sign in and review your first document.

## Prerequisites

Before you begin, make sure you have the following:

- **Node.js** 18 or later
- **LibreOffice** installed on your machine (required for DOCX → PDF conversion)
- A **Supabase** project with Auth and Postgres enabled
- **Cloudflare R2** or another S3-compatible object storage bucket
- At least one AI provider API key (Gemini, Anthropic, or OpenRouter)

## Setup

<Steps>
  <Step title="Clone and install dependencies">
    Clone the Mike repository, then install dependencies for both the backend and frontend from the repository root:

    <CodeGroup>

    ```bash Backend
    npm install --prefix backend
    ```

    ```bash Frontend
    npm install --prefix frontend
    ```

    </CodeGroup>
  </Step>

  <Step title="Create environment files">
    Copy the example environment files to create your local configuration:

    ```bash
    cp backend/.env.example backend/.env
    cp frontend/.env.local.example frontend/.env.local
    ```
  </Step>

  <Step title="Run the database migration">
    Open the **Supabase SQL editor** for your project and run the contents of `backend/migrations/000_one_shot_schema.sql`. This creates all required tables, indexes, row-level security policies, and triggers in a single pass.

    <Info>
      The repository ships a single one-shot schema file (`000_one_shot_schema.sql`). Run this only on a fresh database — it creates all tables from scratch.
    </Info>
  </Step>

  <Step title="Configure environment variables">
    Open `backend/.env` and fill in the values for your services:

    ```bash backend/.env
    # Server
    PORT=3001
    FRONTEND_URL=http://localhost:3000

    # Supabase
    SUPABASE_URL=https://your-project.supabase.co
    SUPABASE_SECRET_KEY=your-supabase-service-role-key

    # S3-compatible storage (e.g. Cloudflare R2)
    R2_ENDPOINT_URL=https://your-account-id.r2.cloudflarestorage.com
    R2_ACCESS_KEY_ID=your-r2-access-key
    R2_SECRET_ACCESS_KEY=your-r2-secret-key
    R2_BUCKET_NAME=mike

    # AI providers — add the keys for the models you want to enable
    GEMINI_API_KEY=your-gemini-key
    ANTHROPIC_API_KEY=your-anthropic-key
    OPENROUTER_API_KEY=your-openrouter-key

    # Email (optional)
    RESEND_API_KEY=your-resend-key
    ```

    Then open `frontend/.env.local` and fill in the Supabase and backend values:

    ```bash frontend/.env.local
    NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
    NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY=your-supabase-anon-key
    SUPABASE_SECRET_KEY=your-supabase-service-role-key
    NEXT_PUBLIC_API_BASE_URL=http://localhost:3001
    ```

    <Note>
      You must provide at least one AI provider key (`GEMINI_API_KEY`, `ANTHROPIC_API_KEY`, or `OPENROUTER_API_KEY`). Mike will only expose models whose keys are configured.
    </Note>
  </Step>

  <Step title="Start the backend and frontend">
    Run both development servers. Start the backend first, then the frontend in a separate terminal:

    <CodeGroup>

    ```bash Backend
    npm run dev --prefix backend
    ```

    ```bash Frontend
    npm run dev --prefix frontend
    ```

    </CodeGroup>

    The backend starts on port `3001` and the frontend on port `3000` by default.
  </Step>

  <Step title="Sign in and review your first document">
    Open [http://localhost:3000](http://localhost:3000) in your browser. Sign up or sign in using Supabase Auth, then create a project, upload a PDF or DOCX file, and start an AI chat to review the document.
  </Step>
</Steps>
